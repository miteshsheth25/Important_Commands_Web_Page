For Database

Database is a collection of inter-related data whichis used to retrieve, insert and delete data efficiently. It is also used to organize the data in the form of table, schema, views and reports.

1. To create a database.

create database if not exists database_name;

2. To use database.

use database_name;

3. To permanently remove the database.

drop database if exists database_name;

4. To see the list of database.

show databases;

For Table

Table is used to organize data in the form of rows and columns and used for both storing and displaying records in the structure format.

5. To create a table.

Different Types of Constraints

Constraints are rules enforced on data column on table They are used to limit the type of data that can go into the table for accuracy and reliability of the data. Constrainnts can be applied at column or table level.

37. Not Null Constraint - Ensures that column cannot nave NUll value.

Create table customers (
  ID int Not Null,
  Name varchar(20) Not Null,
  Age int Not Null,
  Address char(25),
  Salary decimal(18,2),
);

38. Default Constraint - Provides a default value for the column when none is specified.

Create table customers (
  ID int Not Null,
  Name varchar(20) Not Null,
  Age int Not Null,
  Address char(25),
  Salary decimal(18,2) DEFAULT 5000.00,
);

To drop default constraint use :- Alter table customers alter column Salary drop default;

39. Unique Constraint - This prevents two records from having the same values in aparicular column .

Create table customers (
  ID int Not Null,
  Name varchar(20) Not Null,
  Age int Not Null UNIQUE,
  Address char(25),
  Salary decimal(18,2),
)

To add constraint to multiple columns use :- Alter table customers add myUniqueConstraint UNIQUE(Age, Salary);

40. Primary Key - It is a field which uniquely identifies each row/record in the database table. Priomary key column must contain unique value and cannot have      Null value

Create table customers (
  ID int Not Null Primary key,
  Name varchar(20) Not Null,
  Age int Not Null UNIQUE,
  Address char(25),
  Salary decimal(18,2),
)

41. Foreign Key - It is used to link 2 tables togather. It is called refrence key.      Null value

Create table customers (
  ID int NotNull Primary key,
  Name varchar(20) NotNull,
  Age int Not Null,
  Address char(25),
  Salary decimal(18,2),
);

Create table Orders (
  ID int NotNull Primary key,

To drop Foreign constraint use :- Alter table Orders drop Foreign key;

42. Check Constraints - It enables a condition to check the value being entered into the record. If the conditon is false the record is not entered into the table.

Create table customers (
  ID int NotNull Primary key,
  Name varchar(20) NotNull,
  Age int Not Null CHECK(Age>=18),
  Address char(25),
  Salary decimal(18,2),
);

Different Types of Joins clause

Joins clause are used to combine records from 2 or more tables in database. In this atleat 1 data value should be common to each.

44. Inner Join (EQUI JOIN) - It returns rows when there is a match in both tables.

Select ID, Name, Amount, Date from customers inner join orders on customer.ID = orders.customer_ID;

45. Left Join - It returns all rows from left table plus matched values from right table or NULL in case no matching values.

Select ID, Name, Amount, Date from customers left join orders on customer.ID = orders.customer_ID;

46. Right Join - It returns all rows from right table plus matched values from left table or NULL in case no matching values.

Select ID, Name, Amount, Date from customers right join orders on customer.ID = orders.customer_ID;

47. Full Join - It returns all records from right and left table and fill NULL in case no matching values.

Select ID, Name, Amount, Date from customers full join orders on customer.ID = orders.customer_ID;

48. Self Join - In this, join is done to itself as if the table are tow tables, temporarily renaming at least one table.

Select a.ID, a.Name, a.Amount, a.Date from customers1 a, customers1 b where a.customer.ID = b.customers.ID;

49. Union clause / operator - used to combine the results of two or more SELECT statements without returning any duplicate rows.

1. Union:-
             Select ID, Name, Amount, Date from customers Left join orders on customer.ID = orders.customer_ID;
             UNION
             Select ID, Name, Amount, Date from customers Right join orders on customer.ID = orders.customer_ID;

2. Union All:-
             Select ID, Name, Amount, Date from customers Left join orders on customer.ID = orders.customer_ID Union All;
         or
             Select ID, Name, Amount, Date from customers Right join orders on customer.ID = orders.customer_ID Union All;

50. Having clause - It enables you to specify conditions that filter which group results appear in the final result. This clause places conditions on groups created by      GROUP BY clause

select customerName, country, sum(creditlimit) as total_limit from customers group by country having sum(creditlimit) < 100000;

51. View - It is a virtual table based on the result-set. A view contains rows and columns like real table. Thefields in view are from one or more real tables. This       view saved in database for future use

create view credit_limit as
select customerName, country, sum(creditlimit) as total_limit
from customers group by country having sum(creditlimit) < 100000;

SQL Database Transaction

Database Transaction is a single area of the database where multiple data operations are carried out and written as a whole. These operations can be create,
read, update, or delete operations. For a transaction to be successful, it means that every operation carried out has been committed.

52. Start Transaction / Begin - this query triggers the start of a transaction.

Start Transaction;

53. Commit Transaction - this query allows the changes made to the database to become permanent. You can set database to auto-commit changes by using the
     following query.

1. Set autocommit = 1; (to automatically save changes)

2. Set autocommit = 0; (to stop saving changes automatically)

54. Savepoint Transaction - use the SAVEPOINT statement within a transaction to set savepoints. This makes it possible for you to rollback to a specific point in
     the transaction, rather than rolling back the entire transaction.

SAVEPOINT savepointname;

55. Set Transaction - this query allows you to set your commit by enabling the operations to commit automatically or disabling the auto-commit. You can set
     database to auto-commit changes by using the following query.

1. Set autocommit = ON; (to automatically save changes)

2. Set autocommit = OFF; (to stop saving changes automatically)

56. Rollback Transaction - this query allows you to undo the changes you have made to the database, therefore returning the database to its previous
     (last commit) state.

Rollback;
