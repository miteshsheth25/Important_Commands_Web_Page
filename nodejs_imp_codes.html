Most Commonly Used NPM Commands

2. npm init - it initializes a new Node.js project by generating a package.json file containing metadata about the project and its dependencies.

npm init

3. npm install - installs all the dependencies listed in your package.json file. It also allows you to install specific packages globally or locally within a project.

a) Install dependencies :- npm install
b) Install specific package locally :- npm install <package-name></package-name>
c) Install specific package globally :- npm install -g <package-name></package-name>

4. npm uninstall ( or npm rm ) - removes a package from your project. It also removes the dependency from the package.json file.

npm uninstall <package-name>

5. npm update - it updates all installed packages to their latest versions according to the version range specified in the package.json.

npm update

6. npm outdated - using this command, developers can get a list of packages that have newer versions available but are not installed yet in the package.json.

npm outdated

7. npm run - you can run a custom script specified in the scripts section of your package.json. You might use it to test, start servers, or bundle your files.

npm run <script-name>

8. npm list - It displays the tree (a comprehensive list) of installed packages, listing all of your projectâ€™s dependencies in one place.

npm list

9. npm cache clean - it cleans the cache. Cleaning the cache solves issues with package installation.

npm cache clean --force

15. To Connect Node.js with MySQL

a) The following is required to connect Node.js a with MySQL

        1) Node.js version 12 and above
        2) MySQL
        3) MySQL Workbench or CLI-(for database management )

b) Create a new database and table

c) Create a Node.js Project

d) Install mysql12 package

        npm installmysql12

e) Connect Node.js to MySQL using the below script

        const mysql = require('mysql2');

        const db = mysql.createConnection({
                                            host: 'localhost',
                                            user: 'root',         // Replace with your MySQL username
                                            password: '',         // Replace with your MySQL password
                                            database: 'testdb'
                                           });

        db.connect((err) => {
             if (err) {
                  console.error('Database connection failed:', err.message);
                  return;
             }
             console.log('Connected to MySQL database!');
        });

        module.exports = db;

f ) Perform CRUD operations by creating a file app.js as below

        const db = require('./db');

        // Create a new user
        const newUser = { name: 'John Doe', email: 'john@example.com' };

        db.query('INSERT INTO users SET ?', newUser, (err, result) => {
             if (err) throw err;
             console.log('User inserted with ID:', result.insertId);

             // Fetch all users
             db.query('SELECT * FROM users', (err, results) => {
                  if (err) throw err;
                  console.log('All Users:', results);
            });
        });

g) Fixing common MySQL Authentication Error

        if below error is shown

                ER_NOT_SUPPORTED_AUTH_MODE: Client does not support authentication protocol mrequested by server

        Solution: - Run the below code

                1) ALTER USER 'root'@'localhost' IDENTIFIED WITH mysql_native_password BY 'your_password'; FLUSH PRIVILEGES;

                        or

                2) CREATE USER 'nodeuser'@'localhost' IDENTIFIED WITH mysql_native_password BY 'your_password';
                   GRANT ALL PRIVILEGES ON testdb.* TO 'nodeuser'@'localhost';
